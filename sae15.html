Contexte

Cette SAÉ avait pour objectif de développer un outil de traitement automatisé de données issues d’un système de fichiers. L’application permet d’analyser un répertoire, d’identifier les fichiers volumineux et de les représenter graphiquement afin d’aider à la prise de décision.

Le projet repose sur une architecture modulaire séparant trois fonctions principales : l’analyse des fichiers, la visualisation des données et l’action de suppression sécurisée.

L’ensemble est structuré autour d’un échange de données au format JSON entre les différents modules.

Architecture du projet
# Pipeline de traitement des données

Sélection du répertoire
→ analyse du système de fichiers (Python)

analyse_files.py
→ inventaire récursif des fichiers
→ tri par taille décroissante
→ filtrage (seuil + limite)

Export JSON intermédiaire
→ gros_fichiers.json

Interface PyQt5
→ affichage des données
→ visualisation (QPieSeries / QChart)
→ navigation par onglets et actions utilisateur

Script PowerShell
→ suppression sécurisée des fichiers sélectionnés
Mes tâches personnelles

J’ai principalement développé le module d’analyse des fichiers en Python, incluant l’inventaire récursif du système de fichiers et le tri des résultats par taille.

J’ai également participé à la création du format JSON d’échange entre les modules et à l’intégration des données dans l’interface PyQt5 (graphique circulaire).

Enfin, j’ai contribué à la mise en place du script PowerShell permettant la suppression contrôlée des fichiers identifiés.

Preuves de compétences

AC12.04 — Appliquer les bases de l’algorithmique et de la programmation

Conception d’un algorithme de tri et de filtrage de données

J’ai implémenté un algorithme d’inventaire récursif des fichiers, combiné à un tri par taille décroissante et à un filtrage selon des critères définis. Cela m’a permis de transformer un besoin fonctionnel en solution algorithmique structurée.

AC12.05 — Structurer et exploiter des données

Utilisation d’un format d’échange entre modules

J’ai conçu un fichier JSON intermédiaire permettant la communication entre le module d’analyse Python et l’interface graphique PyQt5. Cette approche m’a permis de comprendre l’importance des formats de données dans une architecture logicielle modulaire.

Réflexivité

Cette SAÉ m’a permis de comprendre l’importance de la séparation des responsabilités dans une application. L’analyse, la visualisation et l’action utilisateur doivent être découplées pour garantir un système évolutif.

Le choix du JSON comme format d’échange a été déterminant pour assurer une communication simple entre les modules Python et l’interface graphique.

Enfin, ce projet m’a aidé à progresser en algorithmique, en structuration de données et en conception logicielle.

English Summary

This project consisted of developing a Python application that analyzes filesystem data, sorts files by size, and visualizes the results using a PyQt5 interface.
