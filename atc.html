Contexte du projet

Dans le cadre de la SAÉ 2.03 du BUT Réseaux & Télécommunications, notre groupe de trois étudiants devait développer une application web permettant de gérer des données liées au trafic aérien. L'application devait permettre la consultation, l'ajout, la modification et la suppression des informations concernant les aéroports, les pistes d'atterrissage, les compagnies aériennes, les avions et les vols.

Le projet comprenait également la conception et l'installation d'une base de données relationnelle, le développement d'une interface web avec Django ainsi que le déploiement complet de la solution sur une infrastructure Linux mise en place par notre groupe.

Cette SAÉ m'a permis de mettre en pratique les notions abordées dans les ressources de programmation, bases de données, systèmes d'exploitation, technologies web et travail collaboratif.

Architecture mise en place

L'application a été déployée sur deux machines virtuelles Debian 12 hébergées sous VirtualBox.

  • VM1 (192.168.1.10) : hébergement de l'application Django avec Gunicorn et Nginx.
  • VM2 (192.168.1.20) : serveur MariaDB contenant l'ensemble des données du projet.

Les deux machines communiquaient via un réseau interne VirtualBox tout en conservant un accès Internet grâce à une interface NAT. Cette architecture nous a permis de séparer les services applicatifs et la base de données comme dans une infrastructure réelle.

Développement en groupe avec GitHub et répartition des tâches.

Mes tâches personnelles
  • Création du CRUD complet des aéroports.
  • Création du CRUD complet des pistes d'atterrissage.
  • Développement des vues, formulaires, URLs et templates associés.
  • Validation des données côté serveur avec Django.
  • Création et chargement des fixtures initiales.
  • Installation et configuration de la VM MariaDB.
  • Création de la base de données et gestion des accès distants.
  • Installation et configuration de la VM Django.
  • Mise en place de Gunicorn et Nginx.
  • Configuration réseau entre les deux serveurs.
