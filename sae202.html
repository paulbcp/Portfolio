Contexte

La SAÉ2.02 sort du cadre purement réseau pour entrer dans l’IoT : conception d’une radio connectée sur ESP32 capable de streamer des stations françaises (France Inter, NRJ, Skyrock…) et d’être pilotée à distance via MQTT depuis un broker public.

La version finale (V_PGM_8) gère 7 stations, le volume, les basses, les aigus et un mode de spatialisation audio, entièrement contrôlé via des topics MQTT.

Mes tâches personnelles

J’ai implémenté la communication MQTT sur l’ESP32 (souscription aux topics, parsing des messages, gestion des reconnexions) ainsi que la fonctionnalité de spatialisation audio via le registre SCI_MODE du VS1053. J’ai également participé aux tests de stabilité du firmware en conditions de coupure réseau et de reconnexion au broker.

Architecture MQTT
# Broker : broker.hivemq.com:1883
# Client ID : WebRadio_12

# Topics de commande
webradio/12/station → index station (0–6)
webradio/12/volume → 0–100
webradio/12/bass → registre VS1053
webradio/12/treble → réglage tonalité
webradio/12/spatial → mode EarSpeaker
webradio/12/url → flux direct
webradio/12/stop → arrêt lecture

# Topic état
webradio/12/info ← JSON état actuel
Détail technique — spatialisation VS1053

La spatialisation audio repose sur une écriture directe dans le registre SCI_MODE du VS1053 via SPI, en activant les bits SM_EARSPEAKER_LO et SM_EARSPEAKER_HI.

// VS1053 — configuration SPI
#define SCI_MODE 0x00
#define SM_EARSPEAKER_LO (1 << 4)
#define SM_EARSPEAKER_HI (1 << 7)

// Mode 0 = off | Mode 3 = spatialisation max
// SPI : CS=32, DCS=33, DREQ=15
# Exemple de station radio
[0] France Inter → radiofrance.fr/inter

Réflexivité

Ce projet a marqué ma première vraie expérience de système embarqué : aucune interface graphique, uniquement des logs série et des comportements matériels à interpréter.
